# Public release file policy（公开发行文件策略）

## `temp/` 目录的定位

仓库根目录下的 `temp/` 是**本地审计、验证、合同检查与阶段证据输出**的隔离区（例如 dist 留存审计、图解审计、`post_*` / `worktree` 审计、`*_latest.json` / `*.md`、矩阵或业务阶段结果等）。这些产物用于本地对账与排错，**有保留价值，不等同于可随意丢弃的临时垃圾**。

## 仓库与发行边界

- `temp/**` **不**进入公开 GitHub 仓库（由根目录 `.gitignore` 排除）。
- `temp/**` **不**进入通过 `npm run release:pack` 生成的 GitHub Release 玩家 zip（打包脚本未将该目录列入拷贝清单）。
- 为满足公开首发或发行清单，**不得**删除或移动 `temp/` 内已有文件来「凑合通过」；隔离手段仅限于 **Git 忽略** 与 **发行包排除**。

## 升格为长期基线

若要把 `temp/` 中某一文件提升为团队可引用、路径稳定的正式基线，必须先迁入明确目录，例如：

- `qa/**`
- `validation/baselines/**`（若仓库采用该结构）
- `docs/public/**`

再更新代码或文档中的引用路径；**不得**长期以 `temp/` 作为唯一事实来源。
