# 区域说明（前场竖切）

## 1. 风堤街前场工作竖切（当前边界）

- 当前区域是“前场工作竖切”，不是完整工业区开放。
- 核心作用：
	1. 建立玩家首次劳动/挣钱入口。
	2. 建立分流结构与前场边界认知。
	3. 建立派工入口到执行落点的最小闭环。
- 明确未开放：仓储内部、维修厂内部、燃料库、机房、正式岗位体系。
- 范围说明：本页只描述风堤街工业前场竖切；WEST2 当前骨架已包含 `W2-Spine-0 港口外缘抵达区`，地图已接到港口入口层，且最小 ship/ending 接线已成立，但这些港口口径不属于本页细节范围。港口当前仍不是完整航运系统，未落地范围仍包括完整船表、常规到港循环、通用正式登船流程与更多船型。短工正式 contract 与 Theseus 口径以 `IMPLEMENTATION_REPORT.md`、`运维/基本机制.md`、`docs/timed_location_runtime_contract.md` 为准。

## 2. 四节点职责与链路

### 2.1 风堤街转角

- mapId：`winddyke_street_corner_notice`
- 职责：前场导流锚点，连接诊所侧与工区分流口。
- 状态：前场节点，承载观察与移动，不承载内部工区内容。

### 2.2 工区分流口

- mapId：`industrial_split`
- 职责：当前唯一明确的前场派工入口，承担线路分流。
- 状态：前场节点，承载观察、派工板/签到窗语义、短工入口。

### 2.3 仓储区门口

- mapId：`industrial_warehouse_gate`
- 职责：物流前场边缘节点，承接执行语义与边界说明。
- 状态：相对更开放，但仍为前场边缘，不等于仓储内部作业区。

### 2.4 维修厂门口

- mapId：`industrial_maintenance_gate`
- 职责：门禁与边界说明节点，强调规则与回流。
- 状态：更封闭的前场边缘节点，不承担临工入口。

## 3. 关系图（当前）

- 主链：风堤街转角 <-> 工区分流口。
- 分支：工区分流口 <-> 仓储区门口。
- 分支：工区分流口 <-> 维修厂门口。

## 4. 与系统层的对应关系

- 派工入口职责：工区分流口。
- 执行现场语义：仓储区门口。
- 边界/规则说明：仓储区门口与维修厂门口（语义侧重点不同）。
- 交互系统（观察/询问/短工、job session、inquiry session、inline session 表现）以实现报告为准。
