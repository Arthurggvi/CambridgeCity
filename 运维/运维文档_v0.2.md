### 2026-05-13｜野外系统封板收口：回收物资、随机事件、hover 预览与小地图表现

- **West2 可回收物资已纳入正式物品体系**：`data/items/modules/west2_rescue_supply_items.json` 当前包含 20 个 `material` 物品，均声明 `submission.enabled=true`、`channel:"rescue_station"`、`value` 与 `quality`；品质仅使用 `white/green/blue/pink`，不使用 `iridescent`。背包 truth 仍只保存 `{ itemId, qty }`，不写 `quality/value/submission/category`。
- **野外随机事件仍完全走地图链**：随机事件展示与选择继续使用 `wilderness_event_runtime`，不新增浏览器弹窗、overlay 或 renderer 私有结算；renderer / view model / runtime 不直接改背包或玩家属性。

  - `apply_player_delta`：commit 侧仅结算 `hp` / `stamina`，clamp 到 `0..100`，不支持 `fatigue/satiety/temperatureC/hypothermia` 等额外属性。
- **随机事件奖励不共享池**：奖励必须写在具体事件的具体 outcome 内；不得新增 `west2_common_reward_pool`，不得按品质抽任意物品。当前 West2 事件表包含 `west2_surface_debris_glint_001`、`west2_hidden_snow_hollow_001` 与 6 个良性巡查线事件；恶性 `west2_hidden_snow_hollow_001` 的 hp/stamina 代价由 commit 侧结算。
- **野外移动 hover 控制台为只读预览**：展示方向、耗时、体力与短描述，使用独立控制台式输出栏，宽度随主内容区域变化；不写 `gameState`，不触发 dispatch，不影响移动结算。
- **野外小地图动画为 UI-only 表现层缓存**：world 层平滑移动，arrow 固定中心并平滑旋转；动画缓存不进入存档，不改 VM，不影响 movement resolve/commit。
- **野外工具页关闭修复已落地**：关闭按钮可关闭工具页，并清理遮罩 / blur / inert 残留状态。

### 2026-05-07｜野外向量地图蓝图编辑器（离线作者工具）

- tools/wilderness_area_preview 当前新增蓝图绘制能力，定位为**离线作者工具**。
- 支持：唯一蓝图层、三种混合模式、`terrain_add / cell_subtract / special_map_cell`。
- `terrain_add` 已接入按 `terrainId` 区分的制图符号系统；地貌列表来源仍为 terrain defs（展示层 registry 不替代真值）。
- 蓝图状态只保存 **grid 坐标**；WSAD / pan / zoom / resize 后会重投影并跟随底图移动。
- 默认导出 **`wilderness_blueprint_compact`**（schemaVersion 2）；兼容 **`wilderness_blueprint_delta`**、评审用 **`wilderness_area_merge_preview`**；textarea **导入为蓝图层**（替换当前蓝图）；另附蓝图补丁 DSL **「执行蓝图代码」**（写入 textarea，不写文件）。
- **不写回 data**，不进入 runtime，不写 `world.wilderness`，不接 save/load；导入 / DSL / 绘制 **仅进蓝图层**。
- `tools/wilderness_area_preview/index.html` **由 exporter 生成，禁止手改**。
- 一键覆盖地图通过本地作者服务 apply 管线落地：默认先 `dry-run`，遇到越界坐标走 **HTTP 200 二段确认分支**（不再用 HTTP 400 异常通道）；只有用户确认「允许扩展边界并覆盖」后才会显式 `allowExpandBounds=true` 并允许 `compile_write` 追加 `--allow-expand-bounds`。
- apply 业务失败（`out_of_bounds/compile_write_failed/static_contract_failed`）不得反写为“未发现作者服务”；连接态仅由 discovery/网络失败决定。排错优先看 `debugId` 与 `temp/wilderness_blueprint_apply_debug/<debugId>/` 样本目录。
- static contract 已从 West2 旧内容坐标样本解耦，改为结构/合法性/算法 fixture 合同，避免阻断合法的新蓝图覆盖。
- specialMapCells 作为“特殊地图节点”要正式落地，必须满足：mapId 为真实 `data/maps` 地图 id；占位 `"1"` 禁止落地；一键覆盖时 `compile_write` 必须包含 `--emit-landmarks`；generated landmarks 必须合并进 `AreaSpec.landmarks`；预览页 `vectorVm.mapNodes` 必须包含该节点，页面 marker 才会显示。通过样本：`[-7, 1, "west2_outpost_hub", "前哨出入口"]`。
- 详细口径见：`运维/野外向量地图蓝图编辑器_运维手册_v1.md`、`tools/wilderness_area_preview/README.md`。

- `blockerStops`：时间推进被 blocker 截停。

- 浏览器特有兼容问题（移动端 viewport、输入法、滚动弹性）。
- DOM 点击绑定、按钮命中区域、事件冒泡顺序类问题。
- 渲染层交互异常（如遮罩层层级、过渡动画时序）。

### 2. Node/debug 工具与浏览器 loader 路径差异

- 浏览器侧 loader 运行在 same-origin live server 下，`data/maps/*.json` 之类资源请求必须从仓库根 URL 稳定解析。
- Node/debug 工具（例如 `scene_walker`、若干 probe）运行时经常以 `file:` 协议直接消费模块与数据，不经过浏览器同源 fetch 语义。
- 因此同一份 loader 路径修复，不能只验证浏览器；否则容易出现“浏览器通过，但 Node/file 调试工具 404”或反过来的单边通过。

### 3. loader 在浏览器与 Node 下的兼容策略

- 浏览器下：所有 JSON 请求统一按仓库根 URL 解析，避免 live server 子路径把 `data/maps/menu_main.json`、`data/maps/menu_load.json` 拼到错误地址。
- Node/file 下：当 loader 解析出的资源 URL 为 `file:` 协议时，应改走磁盘读取与 JSON 解析，而不是继续沿用浏览器 fetch 语义。
- 运维回归口径必须明确：浏览器路径可达性与 Node/file 兼容性都是 loader 契约的一部分，修一侧、坏一侧，视为未收口。

### src/save/ - 存档系统

#### save_manager.js - 存档管理器
**职责**：统一存档API
- `saveToSlot(slotId, gameState)` - 保存
- `loadFromSlot(slotId)` - 加载
- `listSlots()` - 列出所有槽位
- `deleteSlot(slotId)` - 删除
- `exportSlot(slotId)` - 导出JSON
- `importToSlot(slotId, jsonString)` - 导入JSON

**规则**：
- ✅ 支持 4 个槽位："auto", 1, 2, 3
- ✅ 主档 + 备份双保险
- ✅ 写入后立即验证
- ✅ 所有操作返回 `{ok: boolean, error?: string}`

#### save_schema.js - 数据结构
**职责**：定义存档格式
- `SAVE_SCHEMA_VERSION` - 当前版本号
- `makeEmptySnapshot(gameState)` - 提取可保存字段；新存档默认不写 `snapshot.player.achievements`
- `sanitizeSnapshot(snap)` - 清理不该保存的字段
- `validateSnapshot(snap)` - 验证完整性；旧档 achievements 字段按 optional legacy field 校验
- `makeSaveFile(slotId, snapshotState, savedAtMs)` - 构建固定 SaveFile
- `validateSaveFile(saveFile)` - SaveFile 外层强校验

**achievements 字段合同**：
- authoritative truth：`CambridgeCity_Profile_Achievements`
- compatibility mirror：`gameState.player.achievements`
- legacy optional field：`snapshot.player.achievements`
- marker：`CambridgeCity_Profile_Achievements_Migrated_v1`
- 旧档 achievements 只允许作为 sanitize / validate 兼容字段与 migration 输入，不得写成新存档真相。

**SaveFile 固定结构（v6）**：
```json
{
  "schemaVersion": 6,
  "savedAt": 1710000000000,
  "slotId": "auto|1|2|3",
  "state": { }
}
```

#### migrations.js - 版本迁移
**职责**：自动升级旧存档
- `migrateSaveFile(saveFile)` - 检测并迁移
- 当前迁移链：`v0 → v1 → v2 → v3 → v4 → v5 → v6`
- `migrateV*toV*()` - 分段迁移函数（按版本递进执行）

#### storage_local.js - 存储适配器
**职责**：封装 LocalStorage
- `read(key)` / `write(key, value)` / `remove(key)`
- 错误处理（空间不足等）

## 常见问题 FAQ

**Q: 为什么不用 Vue/React？**
A: 保持简单，避免框架依赖，方便理解和修改。

**Q: 为什么派生数据不保存到 state？**
A: 派生数据可以随时重新计算，保存会浪费空间且容易不同步。

**Q: 为什么自动存档不是每次操作都触发？**
A: 自动存档采用策略驱动（5/10/30 分钟或 milestone），用于在数据安全与写入频率之间折中，避免每次点击都写 LocalStorage。

**Q: 如何禁用自动存档？**
A: 进入 `menu_settings`，将“自动存档”切换为“关”；该设置保存在 `CambrianCity_Settings_v1`（兼容旧键迁移），不进入存档快照。

**Q: 如何增加存档槽位数量？**
A: 修改 `save_manager.js` 的 `listSlots()` 函数，将循环改为 `for (let slotId = 1; slotId <= 5; slotId++)`。

**Q: 想临时关闭沉浸背景动画怎么做？**
A: 在 `style.css` 注释 `body::after` 的 `animation: atmosphereDrift ...`，或把动画时长改大（例如 60s）。

**Q: 新游戏为什么不再进入 start 地图？**
A: 新游戏入口已切换为诊所病床引导链（`intro_clinic_bed` 系列），旧 `start.json` 已移除，避免双入口分叉。

**Q: 为什么时段切换时会有一次轻微闪动？**
A: 这是 `body.phase-shift` 的跨段反馈，只在“六段切换点”触发一次，不会每分钟触发。

### 6. 禁用事项

- `logLines`
- `actionFeedback.base`
- `uiFeedback`
- `ui.feedback`
- `showNotice`
- `notice`
- `modal`
- `dialog`
- `overlay`
- `records`
- `archive`
- `guide`
- `save/load`

禁止为了单个观察动作修改：

- `src/engine/renderer.js`
- `src/engine/pipeline/dispatch.js`
- `src/engine/pipeline/commit.js`
- `src/engine/pipeline/resolve_handlers/map_handlers_v2.js`

- `src/engine/map_content_v2.js`

- minimap 相关文件
- `style.css`

除非已证明现有地图链能力本身断裂。

### 7. 清理要求

改为纯地图链后，必须清理旧残留：

- `actionFeedback.base`
- `uiFeedback`
- `ui.feedback`
- `effects.push -> logLines` 长文案

允许保留：

- `text`
- `actionLabel`
- `sceneId`
- `edgeId`
- `minutes`
- 普通地图结构字段

### 8. 验收标准

必须同时满足：

- 点击后显示在地图正文区域
- `currentMapId` 不变
- `currentSceneId` 切到目标观察 scene
- 返回动作能回原 scene
- 时间消耗按 edge 生效
- 不出现 modal
- `body` 无 `modal-open`
- 背景不 blur
- 不出现 dimmed
- 不调用 `showNotice`
- 不产生 `report.uiFeedback`
- 不触发 records/archive/guide/save/load
- 不改 overlay/minimap/renderer 壳层

### 9. Cursor 执行口径

本任务是纯地图链任务。

执行顺序：

1. 先对比现有 `TRANSITION + edgeId + scene 正文 + 返回动作` 样本。
2. 只允许改 `data/maps` 与 `data/story/locations` content。
3. 将目标动作改为 `TRANSITION`。
4. 新增 edge，时间消耗写在 edge。
5. 新增目标 scene。
6. 正文写入 `RuntimeText.mapTextByMapId["<mapId>#<sceneId>"].description.base`。
7. 目标 scene 增加返回动作。
8. 删除 `actionFeedback` / `uiFeedback` / `logLines` 长文案残留。
9. 不得改 src、overlay、minimap、records/archive/guide/save/load。
10. 不得使用 `showNotice` / modal。

### 10. 一句话规则

现场观察结果默认落地图 scene；通知、日志、面板都不能冒充地图链。
