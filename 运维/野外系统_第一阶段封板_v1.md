## 3. 关键实现口径（与当前代码一致）

### 3.1 入口与运行时

- 野外运行时地图为 **`wilderness_runtime`**。
- **当前不使用每格一张地图**；坐标真值在 `world.wilderness`（向量格）。
- 当前 runtime **action 固定为 8 个 `WILDERNESS_MOVE` + 1 个 `WILDERNESS_END_SESSION`**（共 9 条）。
- 伊森救援**不新增** `WILDERNESS_CALL_RESCUE`。

### 3.2 移动

- 每格合同口径为 **`metersPerCell: 150`**（见 `west2_old_marker_patrol_line` 区域定义）。
- 八方向移动通过 **vector coordinate** 更新 `x/y` 与 `heading`。
- **斜向移动当前按一格处理**（与正交一致为一步一格）。
- 成功移动链：**`advanceTimeMinutes` → `applyTimeToPlayer` → movement stamina extra cost**（在生存结算之后扣额外体力）。

### 3.3 blocker

- **评估顺序**（概念链，与实现一致）：  
  **boundary → terrain_requirement → terrain_hard → weather_terrain（占位）→ player_state → landmark → normal movement**。
- blocker **命中**时：**不推进时间、不扣 stamina、不改坐标**。
- blocker notice 走**游戏内 dialog**；**不使用** `alert` / `confirm` / `prompt`。

### 3.4 surface

- surface runtime 为**只读派生**；**不写** `world.weather`，**不写** `world.wilderness.surface` 持久字段。
- movement cost 读取 surface 派生的 **`snowDepthMoveMult` / `snowDepthStaminaMult`**（与 `terrain.move` 倍率共同作用）。

### 3.5 probe

- probe 为**只读预览**，不写会话真值。
- **`detectRadius` 仅提示**，不触发进入室内/地标地图。
- **`enterRadius` 命中**才触发 **landmark intercept**（进入对应地图）。

### 3.6 weather forecast

- forecast 基于现有 **weather event 模型同构预测**。
- **不调用** `Math.random`（确定性 RNG）。
- **不推进**真实天气、**不写** `world.weather`。
- **不新增** `blizzard` / `katabatic` / `whiteout` 等**真实** `weatherEventType`；**whiteout** 仅作派生风险/展示层口径。

### 3.7 survival

- **`applyTimeToPlayer`** 为正式生存结算入口。
- **resolver / renderer** 不写 `temperatureC` / `hypothermia` / `hypoStage` / `hp`（写口在 commit 主链）。
- movement **额外** `staminaCost` 在 **survival 之后**扣除。

### 3.8 player_state_block

- 低 HP、严重失温、体力不足等在 **resolve 阶段**阻断可尝试通行的移动。
- **`hypothermia` 为安全指数**：越高越安全，越低越危险。
- **`MIN_HYPOTHERMIA_SAFETY_TO_MOVE = 25`** 为当前「严重失温不宜继续移动」的阻断阈值（见 `wilderness_player_state_blocker.js`）。

### 3.9 landmark

- 当前样板地标：**`maintenance_corridor_entry`**。
- 命中后进入 **`west2_maintenance_corridor_entry`**；返回 **`wilderness_runtime`** 后 **保留同一 `world.wilderness` 会话**（坐标不丢）。

### 3.10 Ethan rescue

- **无**主动求救按钮；**无** `WILDERNESS_CALL_RESCUE`。
- **仅在** `WILDERNESS_MOVE` **成功 commit 且 survival 结算完成后**检测：
  - **stamina** 从 `>0` 到 `<=0`；
  - **fatigue** 从 `>0` 到 `<=0`。
- **`fatigue_zero` 优先于 `stamina_zero`**（同 tick 两者同时成立时按疲劳处理）。
- **`HP <= 0` 不触发**伊森救援（归 DEAD 链）。
- **`satiety` / `temperatureC` / `hypothermia` / `hypoStage` 不参与**伊森救援触发判定。
- **`stamina_zero` 成功**：进入 **`wilderness_ethan_rescue_offer`**，由玩家选择「跟伊森回去」。
- **`fatigue_zero` 成功**：直接进入 **`west2_outpost_rescue_aid_tent_bed`**。
- 救援成功后：约定 **0–100 生存属性 floor 到 20**；**`temperatureC` 仅极端下限保护（≥20），不代表医学意义上的「恢复正常体温」**。
- **`applyEthanRescueRecoveryFloor` 不得改写 `player.meta.sleepEpisode.mode`**（与昏迷演出 `COLLAPSE` 语义分离）。
- **`eventKey`** 防同一事件重复 roll / 重复演出。
- **DEAD 不被 suppress**；**Ethan rescue 与 generic COLLAPSE** 通过 commit report 字段 **互斥**（不依赖 renderer 裁决）。

### 3.11 save/load

- **`world.wilderness` 活跃会话**相关字段经 **sanitize / normalize** 保留（含坐标、`areaId`、`regionId`、`state`、`flags`、`sessionStartedAt` 等，以总验收与 audit 脚本为准）。
- **`RESCUE_PENDING` / `RECOVERED` / `flags`** 经 sanitize/normalize **不变形为非法 state**。
- **本阶段未新增 save schema**（伊森去重等走 `world.wilderness.flags`）。

## 5.1 封板后变更记录：Phase Next A 移动语义局部修订（2026-05-11）

> 本节为**封板后变更记录**，**不改写**第一阶段封板事实；§1 的封板结论、§3 的实现口径、§4 的禁止项继续生效。  
> 本节仅记录在「第一阶段封板（2026-05-04）」之后、围绕「巡查线内强约束 → 通行规则分层」做的局部修订事实与未完成项。

### 5.1.1 已落地事实

- **`activeCellKeys` 不再作为 boundary blocker**：保留为区域表提示字段，由 `queryWildernessCoordinate()` 透出 `inActiveCellMask:boolean`（亦称 `inPatrolCorridor`），blocker / resolver / probe 均不再据此打 `boundary_block`。
- **bounds 外才是真正 boundary**：`queryWildernessCoordinate()` 在 bounds 外才返回 `kind:"boundary"` + `boundaryKind:"out_of_bounds"`；resolver 仅对此类返回打 `boundary_block`。`activeCellKeys` 外但 bounds 内的普通格仅按 terrain 走正常 movement 路径。
- **新增真实地貌硬阻断**：`open_water` 与 `coastal_open_water`（`passability.foot:"hard_block"`、`moveTimeMult/staminaCostMult = Infinity`），仍由既有 `terrain_hard_block` 链路命中。**不**把地图外建为 terrain（无 `off_map_void` terrain）。
- **`blockerStyle` 字段分层**（probe / action VM / local minimap VM 同源）：
  - `"void"`：bounds 外（地图外）；
  - `"sea"`：open_water / coastal_open_water；
  - `"hard_terrain"`：其他硬阻断地貌；
  - `null`：可通行（allowed / slow / conditional）。
- **action VM**：`attachProbesToRuntimeActions()` 给 8 个 `WILDERNESS_MOVE` 动作补 `hidden:boolean` 与 `blockerStyle`；out_of_bounds 与各类 hard block 均 `hidden:true`，slow/conditional/allowed 保持 `hidden:false`。
- **local minimap VM**：在原有 segments / fallbackFrameVisible / emptyBoundaryReason / heading 基础上新增 `neighbors[N|NE|E|SE|S|SW|W|NW] = { kind, terrainId, terrainLabel, blockerStyle }`；`kind ∈ {"terrain","boundary","hard"}`。
- **renderer 只消费 VM 字段**：移动盘按 `entry.action.hidden` 决定是否生成按钮；local minimap 按 `neighbors[*].blockerStyle` 渲染 void / sea / hard_terrain 三种侧面样式。renderer 不再自行 query terrain / blocker。

### 5.1.2 未完成项

- **`wilderness_full_acceptance_check.mjs` 尚未重新清零**：本轮局部合同已通过（见 5.1.3），但 full acceptance 总验收脚本本轮**未重新跑通**，**不得**对外口径写成「Phase Next A 已封板」或「总验收通过」。第一阶段封板（§1）的真值仍以原 2026-05-04 的 15 条 exit 0 为准。
- **drift / 迷路偏航**：仍未实现；VM/合同侧未预留 drift 概率写口，resolver 主链未注入偏航。
- **`lastIncomingDirection` / 来源方向白色高亮**：仍未实现；save/state 范围未扩，下一轮与 drift 一起做。
- **小地图完整玩法子题仍未封板**：本轮仅是既有 local minimap 邻格阻断样式 VM 扩展（neighbors[8] + blockerStyle），不等价于「小地图 UI」子题（§4 禁止项中的小地图 UI 仍未开放为新主线）。
- **WEST2 海岸 terrain authoring 未确认**：本轮只新增 `open_water` / `coastal_open_water` 地貌定义，**未**将区域表上的大范围海岸 zone 改指到新地貌；铺设由后续作者轮单独评审，不得把「新增 terrain 定义」等同于「区域表已铺海岸」。

## 5.2 封板收口补充：UI-only 表现、回收物资与随机事件（2026-05-13）

> 本节记录第一阶段封板后的收口成果，不开放新的业务扩展口。实现以当前代码和合同脚本为准。

### 5.2.1 已落地事实

- **移动 hover 控制台**：方向、耗时、体力与短描述为只读预览，使用独立控制台式输出栏，宽度跟随主内容区域；不写 `gameState`，不触发 dispatch，不影响 movement resolve/commit。
- **小地图动画**：world 层平滑移动，arrow 固定中心并平滑旋转；动画缓存属于 renderer UI-only 表现层，不进入存档，不改 VM，不影响移动结算。
- **野外工具页关闭**：关闭按钮已能关闭工具页，并清理遮罩 / blur / inert 残留。
- **West2 可回收物资**：当前 `data/items/modules/west2_rescue_supply_items.json` 已包含 20 个 `material` 回收物资；均声明 `submission.enabled=true`、`channel:"rescue_station"`、回收 `value` 与 `quality`。当前品质只允许 `white/green/blue/pink`，不使用 `iridescent`。
- **随机事件地图链**：随机事件仍完全走 `wilderness_event_runtime` 地图链；不新增浏览器弹窗、overlay 或 renderer 私有状态结算。
- **`resultIntents` 白名单**：
  - `grant_item`：commit 侧发放物品，只写 inventory truth `{ itemId, qty }`，不写 `quality/value/submission/category`；
  - `apply_player_delta`：commit 侧仅结算 `hp/stamina`，clamp 到 `0..100`，不支持温度、失温、疲劳、饱腹等额外属性。
- **随机事件奖励 authoring 规则**：奖励不共享池，必须写在具体事件 outcome 的 `resultIntents` 内；不得新增公共奖励表，不得按品质抽任意物品。
- **West2 当前事件表**：已包含 `west2_surface_debris_glint_001`、`west2_hidden_snow_hollow_001` 与 6 个良性巡查线事件。恶性 `west2_hidden_snow_hollow_001` 的 hp/stamina 代价已由 commit 侧 `apply_player_delta` 结算。

### 5.2.2 禁止项

- 禁止让 renderer、`wilderness_event_runtime` 或 view model 直接改背包、hp、stamina。
- 禁止新增事件奖励共享池。
- 禁止把 `quality/value/submission` 写入 inventory truth。
- 禁止把小地图动画缓存写入 save/load。
- 禁止把 hover 预览接成移动结算入口。

### 5.2.3 本轮低耗合同

- `node scripts/wilderness_event_contract_check.mjs`
- `node scripts/wilderness_event_runtime_contract_check.mjs`
- `node scripts/wilderness_runtime_ui_contract_check.mjs`
- items db 只读检查：20 个 West2 回收物资均可读取，submission 字段匹配，无 `iridescent`。
